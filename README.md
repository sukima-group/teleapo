## 🆕 v31 の新機能（5つ）— 業界事例(Salesforce/Zoho/ZAC/transcosmos CRM)準拠

### ① 098 竹之内優子 アカウント追加（7人目）
- クライアントSTAFF配列、GASのPIN_MAP / FULL_NAME_MAP / APPOINTER_DRIVE_MAP（takenouchi.sukimagroup@gmail.com / 竹之内_CallRecordings）に追加
- リストシートは初回アップロード時に自動作成されます

### ② コンタクト履歴（CRM核心機能）
- 発信ボタンの近くに「**📚 この番号への過去の架電 N回**」を表示
- タップで過去の応対履歴（日付・結果・メモ・切り返しアウト）を直近5件まで展開
- ハイフン・括弧の表記違いも同一番号として認識
- 「以前も電話したのに…」を防ぎ、引継ぎ・再架電の品質が向上（Salesforce/ZAC方式のタイムライン）

### ③ アポのカレンダー表示
- アポ一覧に**月間カレンダー**を追加（前月/次月ボタン・今日をハイライト）
- アポがある日に「🎯件数」を表示
- 日付をタップするとその日のアポだけに絞り込み（再タップで解除）

### ④ 週別推移（直近8週・月曜始まり）
- 集計画面に週ごとの架電数バー・アポ数・アポ率を表示
- 月別推移と合わせて、短期/中期の傾向分析が可能に

### ⑤ データバックアップ＆復元
- 集計画面に「💾 バックアップ保存」「📂 復元（取込）」ボタンを追加
- 自分の全データ（リスト・履歴・アポ・再架電・NG・目標・シフト等）をJSONファイルで保存
- 端末の故障・機種変更時にファイルから復元可能（別ユーザーのバックアップは警告表示）

## デプロイ手順

### Step 1: GAS を更新（098を使うために必須）
1. hr.ashikaga@gmail.com でログインし https://script.google.com/ を開く
2. 既存「テレアポ管理」の「コード.gs」を **Ctrl+A → Delete** → `gas-code.txt` を貼り付け → **Ctrl+S**
3. 関数プルダウンで **「setupSheets」を選択 → ▶実行**
4. 「デプロイ」→「デプロイを管理」→「✏️編集」→「**新しいバージョン**」→「デプロイ」

### Step 2: GitHub に8ファイルアップロード
https://github.com/sukima-group/teleapo →「Add file」→「Upload files」→ 8ファイル →「Commit changes」

### Step 3: 全端末でキャッシュクリア
- **PC**: F12 → Application → Clear site data → Service Workers → Unregister → Ctrl+Shift+R
- **スマホ**: PWAアイコン長押し→削除 → サイトデータ削除 → 再アクセス → ホーム画面に追加
- ※「更新しますか？」ダイアログが出たら「OK」でも可

### Step 4: 動作確認
- ✅ ログイン画面に緑色の「**v31**」バッジ
- ✅ 管理者のリスト読込プルダウンに「**竹之内優子**」が表示される
- ✅ PIN **098** でログインできる
- ✅ 同じ番号に2回目の架電をすると「📚 この番号への過去の架電」が表示される
- ✅ アポ一覧にカレンダーが表示され、日付タップで絞り込める
- ✅ 集計画面に「📊 週別推移」「💾 データバックアップ」がある

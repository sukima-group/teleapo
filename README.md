## v22 で根本解決した問題

### 報告された症状
- AパソコンのAアカウントで管理者ログイン → リスト読込/削除
- BパソコンのBアカウントでアポインターログイン → Bが過去に設定したリストが残ったまま
- Aで設定/削除した内容が反映されない

### 真の原因 (深堀り調査の結果)

#### 原因1: loadFromGAS が空リストを無視していた
v21までの実装:
```javascript
if(d.list && d.list.length > 0){  // ← 空なら処理されない
  rows = d.list;
}
// → 空の場合は古いローカルデータが残る
```

#### 原因2: clearList がローカルのみ削除でGASに削除を送信していなかった
- 管理者が「リスト削除」 → ローカルだけ削除
- **GASスプレッドシートにはリストが残ったまま**
- 別端末でログインするとGASから古いリスト復活

### v22 の修正

#### ① loadFromGAS: サーバーが正
```javascript
if(d.list !== undefined){
  if(d.list.length > 0){
    rows = d.list;
    // ローカル更新
  } else {
    // GASが空 = ローカルもクリア
    rows = []; cm = null;
    sv("l-"+staff.id, []);
    sv("c-"+staff.id, null);
  }
}
```

#### ② clearList: GASにも削除を送信
```javascript
// 削除確認後、各PINに対して空配列をアップロード
// = サーバーシートをクリア
uploadListInChunks(p.pin, [], function(){...})
```

#### ③ 削除中の進捗UI
削除中は進捗オーバーレイで「削除中 N/M」表示。完了確認後にダイアログを閉じる。

#### ④ 確認ダイアログに警告明示
「⚠️ サーバー側からも削除されます (全端末・全アカウントから消えます)」

## トラブルシューティング

### 削除したのに別端末でリストが残る
- 別端末で「🔄 再同期」ボタンを押す
- またはログアウト→ログイン
- ブラウザキャッシュが原因なら F12 → Application → Storage → Clear site data

### アップロードしたのに別端末で見えない
- GAS再デプロイされているか確認
- Apps Script「実行履歴」でエラー確認
- DevTools(F12) Console でエラー確認

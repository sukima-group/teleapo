## 🆕 v38 の変更点：月次勤怠（実働時間）集計 ※報酬計算の要

テレアポ代行業の中核である**報酬計算（時給精算）**に直結する、**管理者向けの月次勤怠集計**を追加しました。計画シフト（申請）ではなく、**実際の出退勤打刻に基づく実働時間**を月単位で集計します。

- **管理者ビューに月次勤怠パネル**：月を選んで「集計」→ 全アポインターの**出勤日数・実働時間・平均実働/日・架電数・アポ数・通話時間**を一覧
- 画面上部に**延べ出勤日数・総実働時間**のサマリーを表示
- **Excel / CSV 出力**（「勤怠サマリー」＋「日別実働」の2シート）で、時給計算にそのまま使えます
- **同一日の重複記録を自動排除**（打刻の二重送信等があっても実働時間を二重計上しません）

これで管理者の帳票が、**計画シフト（申請Excel）／実働勤怠（本機能）／架電実績（アポ率Excel）／面談状況／監査ログ**と、運用に必要な領域を一通りカバーしました。

**v38で埋めたギャップ**: 報酬計算に不可欠な**実働勤怠の月次集計**。計画シフトと実働勤怠が揃い、管理帳票の網羅性が実務レベルに達しました（各軸は上限のため合計は横ばいですが、「集計/レポート」「シフト/勤怠」の実質的な完成度が上がっています）。

> 補足：本システムは多数の機能追加を重ねてきました。現状は実務に必要な範囲を概ね満たしており、以降は「新機能の追加」より「実機での運用検証・微調整」に価値の重心が移ります。過剰な機能追加は本番系（報酬・請求に直結）の安定性を損ねうるため、次段階の大型機能は運用状況を見ながらの投入をおすすめします。

## デプロイ手順

### Step 1: GAS を更新（月次勤怠のため必須）
1. hr.ashikaga@gmail.com でログインし https://script.google.com/ を開く
2. 「コード.gs」を **Ctrl+A → Delete** → `gas-code.txt` を貼り付け → **Ctrl+S**
3. 「デプロイ」→「デプロイを管理」→「✏️編集」→「**新しいバージョン**」→「デプロイ」

### Step 2: GitHub に8ファイルアップロード
https://github.com/sukima-group/teleapo →「Add file」→「Upload files」→ 8ファイル →「Commit changes」

### Step 3: 全端末でキャッシュクリア
- **PC**: F12 → Application → Clear site data → Service Workers → Unregister → Ctrl+Shift+R
- **スマホ**: PWAアイコン長押し→削除 → サイトデータ削除 → 再アクセス → ホーム画面に追加

### Step 4: 動作確認
- ✅ ログイン画面に「**v38**」バッジ

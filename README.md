## 🆕 v27 の新機能（5つ）

### ① 再架電（かけ直し）ボタンの改善
- 「再架電」を押すと、時間を入れなくても「⏰ 時間未定（後でまた）」ボタンで即保存できるようになりました
- 「本日中にまた」も追加。30分後・1時間後などのクイック設定も継続
- 各結果ボタンに補足説明を追加（不在=誰も出ない / 再架電=かけ直し予定 など）で違いが一目瞭然に

### ② プログレッシブダイヤラー（通話後の自動発信）
- オートコールON時、通話が終わると**カウントダウン後に自動で次の番号へ発信**します
- 「次の発信まで」を 5/10/15/20秒 で設定可能（標準10秒・ラップアップ時間）
- カウントダウン中に「📞 今すぐ発信」「■ 停止」ボタンを表示
- 業界標準のプログレッシブダイヤラー方式（MightyCall/NiCE等）に準拠
- ※スマホOSの仕様上、自動発信はAndroid Chromeで動作。発信時に「通話アプリで開く」が出たら「常時」を選ぶとスムーズです

### ③ 手入力発信のカウント
- ホーム画面に「⌨️ 手入力で発信」欄を追加
- リストにない番号へ発信でき、**本日の発信件数にカウント**＋履歴に記録されます

### ④ 管理者が各アポインターのリスト位置を確認
- 管理者ダッシュボードに「🔄 全員の状況を同期」ボタンを追加
- 各アポインターのカードに「📍 リスト進捗（○/△件・％）」と「最後に架電した会社」を表示
- どこまで架電が進んでいるか一望できます

### ⑤ 退勤通知に最後に架電した会社名
- 退勤時の自動メール（tereapo01.sukima宛）に「📍 最後に架電した会社」を追加
- 退勤レポート・発信件数シートにも「最後の架電会社」列を記録

## デプロイ手順

### Step 1: GAS を更新
1. hr.ashikaga@gmail.com でログインし https://script.google.com/ を開く
2. 既存「テレアポ管理」の「コード.gs」を全削除 → `gas-code.txt` を貼り付け → 保存
3. 関数プルダウンで **「setupSheets」を選択 → ▶実行**（「発信件数」シートの新列を作成）
4. 「デプロイ」→「デプロイを管理」→「✏️編集」→「**新しいバージョン**」→「デプロイ」

### Step 2: GitHub に8ファイルアップロード
https://github.com/sukima-group/teleapo →「Add file」→「Upload files」→ 8ファイル →「Commit changes」

### Step 3: 全端末でキャッシュクリア
- **PC**: F12 → Application → Clear site data → Service Workers → Unregister → Ctrl+Shift+R
- **スマホ**: PWAアイコン長押し→削除 → サイトデータ削除 → 再アクセス → ホーム画面に追加

### Step 4: 動作確認
- ✅ ログイン画面に緑色の「**v27**」バッジ
- ✅ 再架電で「時間未定」で保存できる
- ✅ オートコールONで通話後にカウントダウン→自動発信
- ✅ 手入力発信が件数にカウントされる
- ✅ 管理者ダッシュボードで「全員の状況を同期」→各人のリスト進捗が見える
- ✅ 退勤メールに「最後に架電した会社」が記載される

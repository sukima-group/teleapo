## 🆕 v42 の変更点（ご依頼の3点）

### ① アカウント保護（暗証番号）— 他人のアカウントに入れないように
従来はPIN（091〜098）を知っていれば誰でもそのアカウントに入れました。v42では**各アポインター専用の暗証番号（4桁）**を追加します。

- **初回ログイン**：PIN入力 → 「暗証番号を決めてください」→ 4桁を2回入力して設定（サーバーに**ハッシュ化して保存**・平文は残しません）
- **2回目以降**：PIN入力 → 暗証番号の入力を求められ、**正しい人だけがログイン**できます
- **暗証番号を忘れたら**：管理者画面「🔐 アカウント保護」から**リセット**（次回ログインで再設定）
- **管理者（9999）は暗証番号なしでログイン可**。さらに「○○として操作」から**どのアポインターの画面にも入れます**（ご要望どおり管理者は誰でも入れる）
- オフラインでも、一度ログインした端末なら暗証番号照合が可能（ローカルにハッシュ保持）

### ② 退勤通知が「翌朝まで届かない」問題を修正
原因は、退勤直後にアプリを閉じると、通知（【退勤通知】【発信件数】）が**送信待ちキューに残ったまま**になり、次にアプリを開いた時（＝翌日の出勤時）に初めて送信されていたためです。柏木様で発生していた「前日の通知が翌朝届く」はこれが原因です。

- **アプリを閉じる/バックグラウンドに入る瞬間に `sendBeacon` で未送信をすべて即送信**する仕組みを追加（退勤・出勤・発信件数すべて対象）
- 退勤・出勤の直後にも**即時送信**を実行
- GAS側は**冪等性（重複防止）**があるため、二重送信しても重複メールになりません（12時間の配信済み判定も追加）

### ③ 出勤通知の追加（全アポインター）
**出勤ボタンを押した時**に、`tereapo01.sukima@gmail.com` へ **【出勤通知】** メールが届くようになりました（従来の退勤通知・発信件数に加えて）。これで全アポインターについて、**出勤・退勤の両方**が管理者に通知されます。

## ⚠️ デプロイ

**新エンドポイント（auth_status/auth_set/auth_check/auth_reset/checkin_notify）を使うため、GAS再デプロイ（新しいバージョン）が必須**です。「アカウント認証」シートは初回設定時に自動作成されるため `setupSheets` 再実行は任意。全8ファイルを反映してください。

### デプロイ手順
1. **GAS**：script.google.com →「コード.gs」全消去→ `gas-code.txt` 貼付→保存→「デプロイを管理」→✏️→**新しいバージョン**→デプロイ
2. **GitHub**：sukima-group/teleapo → Upload files → 8ファイル → Commit
3. **全端末**：キャッシュクリア→ホーム画面再追加

### Step 4: 確認手順
1. ログイン画面に「**v42**」
2. **暗証番号**：あるアポインターのPINを入れる → 初回は設定画面 → 4桁2回で設定 → ログイン成功。一度ログアウトし、同じPIN → 暗証番号を求められる → 正しい番号で入れる／違う番号で弾かれることを確認
3. **管理者**：9999 でログイン（暗証番号なしで入れる）→「○○として操作」で各アポインターに入れることを確認
4. **リセット**：管理者「🔐 アカウント保護」で誰かをリセット → その人が次回ログイン時に再設定になることを確認
5. **通知**：あるアポインターで**出勤**を押す → `tereapo01.sukima@gmail.com` に【出勤通知】が届く。**退勤**を押してすぐアプリを閉じる → 【退勤通知】【発信件数】が**その場で**届く（翌朝まで待たない）ことを確認
